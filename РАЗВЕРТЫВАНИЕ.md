# Развертывание

Рекомендуемая схема:

- frontend: Vercel;
- backend: Railway;
- база данных: Railway PostgreSQL.

## 1. Репозиторий GitHub

Проект хранится в GitHub-репозитории. Vercel и Railway используют его для автоматического обновления после отправки изменений.

## 2. Backend на Railway

В Railway нужно создать проект из GitHub-репозитория.

Корневая папка сервиса:

```text
backend
```

В проекте Railway нужно добавить PostgreSQL и настроить переменные окружения backend:

```env
DATABASE_URL=postgresql://...
JWT_SECRET=dlinnaya-sluchaynaya-stroka
FRONTEND_URL=https://adres-frontend-na-vercel.vercel.app
NODE_ENV=production
```

Команда запуска backend:

```bash
npm run db:push && npm start
```

Переменную `PORT` Railway передает автоматически.

## 3. Frontend на Vercel

В Vercel нужно создать проект из того же GitHub-репозитория.

Настройки проекта:

```text
Root Directory: frontend/lms-frontend
Build Command: npm run build
Output Directory: dist
```

Переменная окружения frontend:

```env
VITE_API_URL=https://adres-backend-na-railway
```

После публикации frontend нужно скопировать адрес Vercel в Railway и указать его в `FRONTEND_URL`.

## 4. Обновление проекта

После первичной настройки обновление выполняется стандартной отправкой изменений в GitHub:

```bash
git add .
git commit -m "Краткое описание изменений"
git push
```

После отправки изменений Vercel и Railway автоматически пересоберут проект.
